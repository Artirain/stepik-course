# Claude Code on the Web — браузерная версия

!!! info "Что ты узнаешь"
    - Что такое Claude Code on the Web и как получить доступ
    - Чем браузерная версия отличается от CLI
    - Какие возможности доступны, а какие — нет
    - Когда использовать веб-версию, а когда — терминал

## Введение

Claude Code изначально создавался как CLI-инструмент для терминала. Но не у всех есть возможность (или желание) работать в командной строке. Для таких случаев Anthropic предоставляет Claude Code on the Web — браузерную версию, доступную через claude.ai.

Это тот же агент, но работающий прямо в браузере. Без установки, без терминала, без настройки окружения.

## Как получить доступ

Claude Code on the Web доступен через интерфейс claude.ai. Для работы тебе нужен аккаунт с подпиской Pro или Team.

Порядок действий:

1. Перейди на [claude.ai](https://claude.ai)
2. Войди в свой аккаунт
3. В интерфейсе чата выбери режим Claude Code
4. Подключи свой проект (например, репозиторий с GitHub)

После этого ты можешь работать с кодом прямо в браузере — задавать промпты, редактировать файлы, запускать команды.

## Что умеет веб-версия

Браузерная версия предоставляет ключевые возможности агента:

- **Чтение и редактирование файлов** — Claude видит структуру проекта и может вносить изменения
- **Выполнение команд** — запуск тестов, сборки, линтинга
- **Работа с Git** — коммиты, просмотр diff, работа с ветками
- **Мультифайловые изменения** — рефакторинг, затрагивающий несколько файлов
- **Интеграция с GitHub** — подключение репозиториев напрямую

## Ограничения по сравнению с CLI

Несмотря на удобство, у веб-версии есть ограничения:

| Возможность | CLI | Web |
|---|---|---|
| Работа с локальными файлами | Да | Через подключённый репозиторий |
| CLAUDE.md | Полная поддержка | Поддержка |
| MCP-серверы | Полная настройка | Ограниченная |
| Hooks | Да | Нет |
| Headless-режим | Да | Нет |
| Кастомные slash-команды | Да | Ограниченно |
| Работа без интернета | Да (с API-ключом) | Нет |
| Скорость отклика | Зависит от сети | Зависит от сети |

!!! warning "Важное ограничение"
    Веб-версия не имеет прямого доступа к твоей файловой системе. Она работает с облачной копией репозитория. Это значит, что ты не можешь использовать локальные инструменты, установленные на твоей машине.

## Когда выбрать Web, а когда — CLI

**Используй Web, если:**

- Ты хочешь быстро попробовать Claude Code без установки
- Работаешь с чужого компьютера
- Нужно показать коллеге, как работает агент
- Проект уже на GitHub и не требует локальных инструментов

**Используй CLI, если:**

- Тебе нужен полный контроль над окружением
- Используешь MCP-серверы или hooks
- Работаешь с приватными файлами, которые не хочешь загружать в облако
- Нужна автоматизация (CI/CD, скрипты)
- Работаешь в монорепозитории с кастомной настройкой

## Совместное использование

Веб и CLI не конкурируют друг с другом. Можно использовать оба варианта:

- Дома — CLI с полной настройкой
- В дороге — Web через браузер на планшете
- На код-ревью — Web для быстрого просмотра и анализа
- На проде — CLI с headless-режимом для автоматизации

## Практика

1. Зайди на claude.ai и открой режим Claude Code
2. Подключи любой свой GitHub-репозиторий
3. Попроси Claude Code объяснить структуру проекта
4. Попробуй внести небольшое изменение и сравни опыт с CLI

## Итоги

- Claude Code on the Web — браузерная версия агента, доступная через claude.ai
- Поддерживает основные функции: чтение/редактирование файлов, Git, выполнение команд
- Не поддерживает hooks, headless-режим и полную настройку MCP
- CLI остаётся более мощным инструментом для продвинутых сценариев
- Оба варианта можно комбинировать в зависимости от ситуации
